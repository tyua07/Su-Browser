webdriver​

webdriver 可以用来判断当前网页操作是否使用了自动化工具。

警告

该参数您应该永远把它设置成 false。

注入说明​

注入属性名称类型选项示例
navigator.webdriver-statusint1：true；0：false0

User-Agent​

User-Agent 里包括但不仅限于：浏览器的版本、操作系统平台、操作系统版本、设备型号等信息。

tip

在浏览器的 Client Hints 属性了也包含了 UA 这类信息，所以设置的时候需要配套。如果您有需求，我们内部所提供的的解决方案已经自动化适配。

注入说明​

注入属性名称类型示例
navigator.user-agentstringMozilla/5.0 (Linux; Android 9; vivo X22A Build/PKQ1.180819.001; wv) AppleWebKit/537.36 (KHTML, like Gecko) Version/4.0 Chrome/88.0.3440.91 Mobile Safari/537.36

自动修正版本号​

我们有一个注入参数（version-info.number）是用来修改浏览器的版本号，当您的注入的 UA 版本号与浏览器版本号不一致时，如果该属性的值是 true，则会自动把 UA 里的版本修正为浏览器版本。

例如：UA 信息是 Mozilla/5.0 (Linux; Android 9; vivo X22A Build/PKQ1.180819.001; wv) AppleWebKit/537.36 (KHTML, like Gecko) Version/4.0 Chrome/88.0.3440.91 Mobile Safari/537.36，浏览器的版本号是 101.0.3440.91，则注入程序会自动把 UA 里的版本号修正成 101.0.3440.91。

实际结果： Mozilla/5.0 (Linux; Android 9; vivo X22A Build/PKQ1.180819.001; wv) AppleWebKit/537.36 (KHTML, like Gecko) Version/4.0 Chrome/101.0.3440.91 Mobile Safari/537.36

tip
  1. 自动修正版本号不仅仅是修改了 UA 里的版本号，对应的 Client Hints 里的也会修改。
  2. 设置了该值后，您可以用一个 UA 创建出多个浏览器版本的 UA，再也不用考虑浏览器环境里版本不一致问题。

注入说明​

注入属性名称类型选项示例
navigator.user-agent-auto-matchint1：true；0：false1

平台​

platform 表示当前浏览器操作系统。

注意

注意：如果您想在 PC 端模拟移动端浏览器环境，请把 platform 修改成对应平台。

@unique
class Platform(str, Enum):
    LINUX_ARMV8L = "Linux armv8l"
    LINUX_ARMV7L = "Linux armv7l"
    LINUX_ARMV6L = "Linux armv6l"
    LINUX_AARCH64 = "Linux aarch64"
    LINUX_X86_64 = "Linux x86_64"
    LINUX_I686 = "Linux i686"
    MACINTEL = "MacIntel"
    WIN32 = "Win32"
    WIN64 = "Win64"
    WINCE = "WinCE"
    SUNOS = "SunOS"
    IPHONE = "iPhone"
    IPOD = "iPod"
    IPAD = "iPad"

注入说明​

注入属性名称类型示例
navigator.platformstring"Linux armv8l"
